# CS-330

## Reflection

When it comes to designing software, I usually start by trying to understand what the final result should look like before worrying too much about the code itself. For this project, that meant thinking about how I wanted the scene to look visually and how different objects shoudl beheve before actually implementing them. One design skill I improved on was breaking down a larger idea into smaller parts, like separating the scene into individual objects and handling lighting, textures, and transformations step by step. My design process was pretty iterative. I would build something simple first, test it, and then gradually add more detail or fix issues as they came up. That approach is something I can definitely carry into future work because it makes complex projects feel more manageable.

When developing programs, I tend to focus on getting a working version first and then refining it over time. During this project, I relied a lot on iteration. Like I would adjust object placement, lighting, or textures multiple times until it looked right. I also got more comfortable debugging issues related to rendering and understanding how small changes in the code could affect the entire scene. Compared to earlier milestones, my approach became more structured. Instead of guessing, I started making more intentional changes and testing them one at a time, which made it easier to track what was working and what was not.

Computer science as a whole is important for my long-term goals because I want to move into a filed related to information security, and having a strong technical foundation is necessary for that. Even though computational graphics is not directly related to security, it helped me build problem solving skills and gave me a better understanding of how systems process and display data. Working with graphics also introduced me to concepts like transformations, coordinate systems, and rendering pipelines, which are useful in understanding how software operates at a deeper level.

In terms of my educational pathway, this project helped me get more comfortable wokring with more complex code and understanding how different components come together to form a complete program. Professionally, these skills can still be useful even outside of graphics. Being able to design, build, and refine a system step by step is something that applies to many areas in computer science. It also gave me experience with thinking about user interaction and visual feedback, which can be important when building tools or interfaces in the future.
